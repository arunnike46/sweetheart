# SWEET HEART – Artisan Cake Shop

This repository contains the migrated and updated codebase for the **SWEET HEART** project. The project has been successfully converted from a Java Spring Boot & local MySQL database architecture to a modern, lightweight, serverless stack.

## Architecture Migration & Implementation Plan

### 1. Frontend
- **Tech Stack**: HTML5, Vanilla CSS, Vanilla JavaScript.
- **Dynamic Configuration**: The frontend (`FrontEnd/main.js`) dynamically detects the host environment. If run locally (`localhost` or `127.0.0.1`), it points to the local FastAPI dev server (`http://localhost:8080/api`). For production, it connects directly to the Vercel-deployed server.
- **Deployment**: Hosted on **GitHub Pages**.

### 2. Backend
- **Tech Stack**: **Python 3.9+** using the **FastAPI** framework.
- **Vercel Serverless Integration**: Reorganized using Vercel's serverless structure (`api/index.py` + `vercel.json` routing configuration).
- **Features Preserved**:
  - Auto-generation of unique 6-digit order verification PINs.
  - Calculation of days left until delivery and urgency colors (`rb`, `rr`, `ry`, `rg`).
  - Validation of delivery date (must be at least 1 day in the future).
  - Secure shopkeeper dashboard login endpoint.
- **Deployment**: Hosted on **Vercel**.

### 3. Database
- **Tech Stack**: **Supabase (PostgreSQL)**.
- **Integration**: Leverages the official `supabase-py` client SDK. Communication is secured using environment variables. Row Level Security (RLS) is configured to safely permit inserts from clients while reserving administration capabilities (reads, edits, deletes) for the authenticated backend.

## Step-by-Step Deployment Instructions

### Step 1: Database Setup (Supabase)
1. Go to [Supabase](https://supabase.com/) and log in or register.
2. Click **New Project**, pick your organization, set a project name, database password, and click **Create new project**.
3. Once the database is ready, navigate to the **SQL Editor** tab from the left sidebar.
4. Click **New query**, paste the SQL schema found in [DataBase/schema.sql](file:///f:/SWEET%20HEART/DataBase/schema.sql):
   ```sql
   CREATE TABLE IF NOT EXISTS bookings (
       id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
       name VARCHAR(100) NOT NULL,
       phone VARCHAR(20) NOT NULL,
       cake_name VARCHAR(100) NOT NULL,
       required_date DATE NOT NULL,
       required_time TIME WITHOUT TIME ZONE NOT NULL,
       query TEXT,
       pin VARCHAR(10) UNIQUE NOT NULL,
       created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL
   );
   ALTER TABLE bookings ENABLE ROW LEVEL SECURITY;
   CREATE POLICY "Allow public insert" ON bookings FOR INSERT WITH CHECK (true);
   ```
5. Click **Run** to execute the query.
6. Retrieve API keys and URLs:
   - Navigate to **Project Settings** (gear icon) -> **API**.
   - Copy the **Project URL** (used as `SUPABASE_URL` env variable).
   - Copy the **service_role** API key (used as `SUPABASE_KEY` env variable so the backend can bypass RLS).

---

### Step 2: Backend Deployment (Vercel)
1. Install the Vercel CLI tool:
   ```bash
   npm install -g vercel
   ```
2. Navigate to your backend directory:
   ```bash
   cd BackEnd
   ```
3. Run the initial deployment configuration:
   ```bash
   vercel
   ```
   - Log in if prompted.
   - Link to a new project: `Yes`
   - Project Name: `sweetheart-backend-api`
   - Set root directory to `./`
4. Set up the Environment Variables in the Vercel Dashboard:
   - Go to your project on Vercel.
   - Navigate to **Settings** -> **Environment Variables**.
   - Add the following variables:
     - `SUPABASE_URL` = *(Your Supabase URL)*
     - `SUPABASE_KEY` = *(Your Supabase service_role key)*
     - `APP_SHOPKEEPER_PASSWORD` = `102003` (or your preferred dashboard passcode)
5. Build and deploy to production:
   ```bash
   vercel --prod
   ```
6. Note the output URL (e.g. `https://sweetheart-backend-api.vercel.app`).

---

### Step 3: Frontend Update & Deployment (GitHub Pages)

1. Open [FrontEnd/main.js](file:///f:/SWEET%20HEART/FrontEnd/main.js) and update the `DEPLOYED_API_BASE` URL at the top with your new Vercel backend URL (ensure it includes the protocol and `/api` suffix):
   ```javascript
   const DEPLOYED_API_BASE = "https://sweetheart-backend-api.vercel.app/api";
   ```
2. Open your terminal in the frontend directory:
   ```bash
   cd ../FrontEnd
   ```
3. Initialize Git, commit, and push your changes to GitHub:
   ```bash
   git init
   git add .
   git commit -m "Configure and deploy Sweet Heart frontend"
   git remote add origin https://github.com/YOUR_GITHUB_USERNAME/sweetheart.git
   git branch -M main
   git push -u origin main
   ```
4. Configure GitHub Pages:
   - Go to your repository on GitHub.
   - Click on **Settings** -> **Pages**.
   - Under **Build and deployment**, select **Deploy from a branch** and set the branch to `main`.
   - Click **Save**.

Your application is now fully deployed!
- **Frontend URL**: `https://YOUR_GITHUB_USERNAME.github.io/sweetheart/`
- **Backend URL**: `https://sweetheart-backend-api.vercel.app/api`
